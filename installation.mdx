---
title: 'Install SDRTrunk on Windows, Linux, or macOS with drivers'
description: 'Download the correct SDRTrunk release for your operating system, install any required drivers, and verify the application launches successfully.'
---

SDRTrunk is distributed as a self-contained zip archive that includes a bundled Java runtime. You do not need to install a separate JDK. Download the archive for your operating system, extract it, and run the launcher.

## System requirements

Before you download, confirm your system meets the minimum requirements:

| Component | Requirement |
|---|---|
| Operating system | Windows 64-bit, Linux 64-bit, or macOS 64-bit 12.x or higher |
| CPU | 4-core processor |
| RAM | 8 GB preferred; 4 GB may work for light use |
| USB port | Available USB 2.0 or 3.0 port for your SDR device |

## Choosing a release

All releases are published on the [SDRTrunk GitHub releases page](https://github.com/DSheirer/sdrtrunk/releases).

<AccordionGroup>
  <Accordion title="Final releases (stable)">
    Final releases are marked with a version number such as `0.6.1`. These have been tested and are the recommended starting point for new users. Look for the release labeled **(final)** in the release notes.
  </Accordion>
  <Accordion title="Beta releases (pre-release testing)">
    Beta releases, for example `0.6.2-beta-1`, are undergoing active bug testing before final release. They are generally usable but may contain known issues.
  </Accordion>
  <Accordion title="Alpha releases (feature preview)">
    Alpha releases are early development previews. They are likely to contain bugs and unexpected behavior. Use these only if you want to preview unreleased features.
  </Accordion>
  <Accordion title="Nightly builds (latest commits)">
    The nightly release contains the most recent code changes and is updated whenever new commits are pushed. It is not truly "nightly" — it reflects the current state of the codebase. Always back up your playlist files before switching to a nightly build.
  </Accordion>
</AccordionGroup>

Download the zip file whose name matches your OS and CPU architecture. For example:

- `sdrtrunk-linux-x86_64-v0.6.1.zip` — Linux on an Intel/AMD 64-bit CPU
- `sdrtrunk-linux-aarch64-v0.6.1.zip` — Linux on a 64-bit ARM CPU
- `sdrtrunk-osx-aarch64-v0.6.1.zip` — macOS on Apple Silicon (M1/M2/M3/M4)
- `sdrtrunk-osx-x86_64-v0.6.1.zip` — macOS on Intel
- `sdrtrunk-windows-x86_64-v0.6.1.zip` — Windows on an Intel/AMD 64-bit CPU

## Installation steps

<Tabs>
  <Tab title="Windows">
    <Steps>
      <Step title="Download the Windows zip">
        Go to the [releases page](https://github.com/DSheirer/sdrtrunk/releases) and download `sdrtrunk-windows-x86_64-v<version>.zip` (or the `aarch64` variant for ARM-based Windows devices).
      </Step>
      <Step title="Extract the archive">
        Right-click the zip file and select **Extract All**, or use your preferred archive tool. Extract it to a location you can find easily, such as `C:\sdrtrunk`.
      </Step>
      <Step title="Install USB drivers for RTL-SDR (if applicable)">
        Windows requires a WinUSB driver for RTL-SDR devices. SDRTrunk does not install drivers automatically.

        1. Download [Zadig](https://zadig.akeo.ie/) from zadig.akeo.ie.
        2. Plug in your RTL-SDR.
        3. Open Zadig, select **Options > List All Devices**, then select your RTL-SDR device.
        4. Choose **WinUSB** as the driver and click **Install Driver**.

        <Note>
          If you have already installed RTL-SDR drivers for another application such as SDR#, they may already be compatible. SDRPlay users must install the [SDRplay API](https://www.sdrplay.com/softwarehome/) separately.
        </Note>
      </Step>
      <Step title="Launch SDRTrunk">
        Inside the extracted folder, open the `bin` directory and double-click `sdrtrunk.bat`. The main window should appear within a few seconds and the tuner manager will automatically detect connected SDR hardware.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Linux">
    <Steps>
      <Step title="Download the Linux zip">
        Go to the [releases page](https://github.com/DSheirer/sdrtrunk/releases) and download the zip that matches your CPU architecture (`x86_64` for Intel/AMD, `aarch64` for ARM).
      </Step>
      <Step title="Extract the archive">
        Open a terminal in the directory where you downloaded the file and run:

        ```bash
        unzip sdrtrunk-linux-x86_64-v<version>.zip -d ~/sdrtrunk
        ```

        Replace the filename with the actual version you downloaded.
      </Step>
      <Step title="Make the launcher executable">
        ```bash
        chmod +x ~/sdrtrunk/bin/sdrtrunk
        ```
      </Step>
      <Step title="Configure USB access (RTL-SDR)">
        On most Linux distributions, RTL-SDR devices work without installing a separate driver. However, you may need to add a udev rule so your user account can access the device without `sudo`.

        Create a file at `/etc/udev/rules.d/20-rtlsdr.rules` with the following content:

        ```text
        SUBSYSTEM=="usb", ATTRS{idVendor}=="0bda", ATTRS{idProduct}=="2838", GROUP="plugdev", MODE="0666"
        ```

        Then reload udev rules:

        ```bash
        sudo udevadm control --reload-rules
        sudo udevadm trigger
        ```

        Log out and back in, or add your user to the `plugdev` group:

        ```bash
        sudo usermod -aG plugdev $USER
        ```
      </Step>
      <Step title="Launch SDRTrunk">
        ```bash
        ~/sdrtrunk/bin/sdrtrunk
        ```

        The main window appears and the tuner manager begins scanning for connected SDR hardware.
      </Step>
    </Steps>
  </Tab>
  <Tab title="macOS">
    <Steps>
      <Step title="Download the macOS zip">
        Go to the [releases page](https://github.com/DSheirer/sdrtrunk/releases) and download the zip for your Mac. Choose `osx-aarch64` for Apple Silicon (M1 and later) or `osx-x86_64` for Intel Macs.
      </Step>
      <Step title="Extract the archive">
        Double-click the downloaded zip in Finder, or use Terminal:

        ```bash
        unzip sdrtrunk-osx-aarch64-v<version>.zip -d ~/sdrtrunk
        ```
      </Step>
      <Step title="Install libusb">
        SDRTrunk requires `libusb` to communicate with USB SDR devices on macOS. Install it using Homebrew:

        ```bash
        brew install libusb
        ```

        If you do not have Homebrew, install it from [brew.sh](https://brew.sh).
      </Step>
      <Step title="Make the launcher executable">
        ```bash
        chmod +x ~/sdrtrunk/bin/sdrtrunk
        ```
      </Step>
      <Step title="Launch SDRTrunk">
        ```bash
        ~/sdrtrunk/bin/sdrtrunk
        ```

        macOS may display a security dialog the first time. Go to **System Settings > Privacy & Security** and click **Open Anyway** to allow the application to run.
      </Step>
    </Steps>

    <Warning>
      **macOS Tahoe 26.x users:** Changes to USB support in macOS Tahoe 26.x prevent SDRTrunk from launching with the standard `libusb` release. You must install `libusb` from the HEAD branch and create a symbolic link manually.

      ```bash
      brew install libusb --HEAD
      sudo mkdir -p /opt/local/lib
      ```

      Find the path where Homebrew installed the HEAD build, for example `/opt/homebrew/Cellar/libusb/HEAD-9ceaa52/lib/libusb-1.0.0.dylib`, then create the symbolic link:

      ```bash
      sudo ln -s /opt/homebrew/Cellar/libusb/HEAD-9ceaa52/lib/libusb-1.0.0.dylib \
          /opt/local/lib/libusb-1.0.0.dylib
      ```

      Use a nightly build of SDRTrunk, which includes an updated `usb4java` native library for Tahoe on ARM processors. Note that USB SDR tuner access may still have issues on Tahoe.
    </Warning>
  </Tab>
</Tabs>

## Verifying the installation

When SDRTrunk starts successfully, you should see:

1. The main window opens with a spectral display panel at the top and a controller panel below.
2. The title bar shows **SDRTrunk** followed by the name of any automatically detected tuner.
3. If a compatible SDR is connected, the spectral display shows a live FFT waterfall.

If the spectral display is blank and no tuner appears in the title bar, check that your SDR device is connected, drivers are installed (Windows), and USB permissions are configured (Linux).

<Tip>
You can switch between tuners at any time using **View > Tuners** in the menu bar.
</Tip>

| Hardware | Windows | Linux | macOS |
|---|---|---|---|

| HackRF One | Zadig WinUSB driver required | udev rule for non-root access | libusb via Homebrew |
| Airspy | Zadig WinUSB driver required | udev rule for non-root access | libusb via Homebrew |
| SDRplay RSP series | SDRplay API installer required | SDRplay API installer required | SDRplay API installer required |
| Funcube Dongle | Standard USB audio driver | Standard USB audio driver | Standard USB audio driver |
