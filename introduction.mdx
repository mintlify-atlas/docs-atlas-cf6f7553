SDRTrunk is a cross-platform Java application for decoding, monitoring, recording, and streaming trunked mobile and related radio protocols using Software Defined Radio (SDR) hardware. It runs on 64-bit Windows, Linux, and macOS, and bundles its own Java runtime so you do not need to install a JDK separately.

## What you can do with SDRTrunk

SDRTrunk is used by radio enthusiasts, amateur radio operators, and researchers to receive and decode digital and analog radio traffic in real time. Common use cases include:

- **Public safety monitoring** — follow P25 trunked systems used by police, fire, and EMS agencies
- **Amateur radio** — decode DMR repeaters and simplex channels
- **Research and education** — capture and analyze radio protocol data for study or development

## Supported protocols

SDRTrunk can decode the following protocols:

| Protocol | Description |
|---|---|
| APCO-25 Phase 1 | Digital trunked radio used widely by public safety (P25) |
| APCO-25 Phase 2 | TDMA variant of P25 with higher spectral efficiency |
| DMR | Digital Mobile Radio, used in commercial and amateur radio |
| MPT-1327 | Analog trunked radio protocol used in many countries |
| LTR | Logic Trunked Radio, a simple analog trunked protocol |
| LTR-Net | Networked variant of LTR |
| NBFM | Narrowband FM analog voice |
| MDC-1200 | Motorola signaling protocol over analog FM |
| Fleetsync | Kenwood fleet management signaling |
| Passport | EF Johnson trunked radio protocol |
| AM | Amplitude modulation (aircraft and other AM voice) |

## Key capabilities

**Polyphase channelizer** — SDRTrunk uses a polyphase channelizer to extract many channels simultaneously from a single wideband IQ stream. One RTL-SDR can cover an entire trunked system band and decode multiple active channels at once.

**Real-time spectral display** — the main window shows a live FFT spectrum and scrolling waterfall for the active tuner. You can click a signal in the spectrum to inspect it directly.

**Alias management** — assign names, colors, and icons to talkgroup IDs and radio IDs so decoded traffic appears with human-readable labels instead of raw numbers.

**Audio recording** — automatically record decoded audio to MP3 or WAV files, organized by date and alias.

**Live streaming** — stream audio to Broadcastify, Icecast, and other streaming servers using the built-in audio streaming manager.

**Playlist editor** — all channel configurations, aliases, and streaming destinations are stored in a playlist file that you can save, reload, and share.
