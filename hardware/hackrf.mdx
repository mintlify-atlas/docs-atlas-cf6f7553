The HackRF is an open-source Software Defined Radio platform made by Great Scott Gadgets. SDRTrunk supports three HackRF hardware variants and uses them in receive-only mode, despite the hardware being capable of both transmitting and receiving.

## Supported models

| Model | SDRTrunk identifier | Notes |
|-------|---------------------|-------|
| HackRF One | `HACKRF_ONE` | Current production model |
| HackRF Jawbreaker | `HACKRF_JAWBREAKER` | Original prototype hardware |
| RAD1O | `HACKRF_RAD1O` | CCC badge hardware, HackRF-compatible |

SDRTrunk automatically identifies which board is connected by querying the **Board ID** register over USB at startup.

## Frequency range

HackRF covers **10 MHz to 6 GHz** as configured in SDRTrunk (the hardware itself is specified down to 1 MHz, but SDRTrunk sets a 10 MHz software minimum). This gives it the widest frequency range of any device SDRTrunk supports. The usable bandwidth is approximately 90% of the configured sample rate, with a small unusable region around the DC center.

<Note>
  HackRF is a half-duplex device — it can transmit or receive, but not both simultaneously. SDRTrunk places the device into receive mode at startup and returns it to off mode when stopped.
</Note>

## Sample rates

SDRTrunk supports the following HackRF sample rates. Each rate is paired with a matching baseband filter that is applied automatically:

| Sample rate | Baseband filter |
|-------------|----------------|
| 1.750 MHz | 1.75 MHz |
| 2.500 MHz | 2.50 MHz |
| 3.500 MHz | 3.50 MHz |
| **5.000 MHz** | **5.00 MHz (default)** |
| 5.500 MHz | 5.50 MHz |
| 6.000 MHz | 6.00 MHz |
| 7.000 MHz | 7.00 MHz |
| 8.000 MHz | 8.00 MHz |
| 9.000 MHz | 9.00 MHz |
| 10.000 MHz | 10.00 MHz |
| 12.000 MHz | 12.00 MHz |
| 14.000 MHz | 14.00 MHz |
| 15.000 MHz | 15.00 MHz |
| 20.000 MHz | 20.00 MHz |

<Tip>
  5 MHz is the default sample rate and a practical starting point for most trunked radio monitoring. Use higher rates when you need to capture a wider slice of spectrum.
</Tip>

## Driver installation

<Tabs>
  <Tab title="Windows">
    HackRF requires the **WinUSB** driver on Windows. Use the Zadig tool to install it.

    <Steps>
      <Step title="Download Zadig">
        Download Zadig from [zadig.akeo.ie](https://zadig.akeo.ie) and run it — no installation is needed.
      </Step>
      <Step title="Plug in HackRF">
        Connect your HackRF device. Windows may attempt to install a driver automatically; let it complete.
      </Step>
      <Step title="List all devices in Zadig">
        Open the **Options** menu and check **List All Devices**.
      </Step>
      <Step title="Select HackRF One">
        Choose **HackRF One** (or your specific model) from the device dropdown.
      </Step>
      <Step title="Install WinUSB">
        Confirm the target driver shows **WinUSB** and click **Install Driver** (or **Replace Driver** if another driver is already installed).
      </Step>
      <Step title="Launch SDRTrunk">
        Start SDRTrunk. Your HackRF device should appear in the tuner manager.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Linux">
    Most Linux distributions include HackRF support via the `hackrf` package. Install it to get the required udev rules and library.

    <Steps>
      <Step title="Install the hackrf package">
        Install the package for your distribution:

        ```bash
        # Debian/Ubuntu
        sudo apt install hackrf

        # Fedora
        sudo dnf install hackrf

        # Arch Linux
        sudo pacman -S hackrf
        ```
      </Step>
      <Step title="Add udev rules">
        The `hackrf` package installs udev rules automatically. If you installed from source, copy the rules manually:

        ```bash
        sudo cp /path/to/hackrf/host/libhackrf/53-hackrf.rules /etc/udev/rules.d/
        sudo udevadm control --reload-rules
        ```
      </Step>
      <Step title="Add your user to the plugdev group">
        ```bash
        sudo usermod -aG plugdev $USER
        ```

        Log out and back in for the change to take effect.
      </Step>
      <Step title="Verify the device is accessible">
        Run `hackrf_info` to confirm the device is detected before launching SDRTrunk:

        ```bash
        hackrf_info
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="macOS">
    Install HackRF tools and the libhackrf library using Homebrew.

    <Steps>
      <Step title="Install Homebrew">
        If you do not have Homebrew, install it from [brew.sh](https://brew.sh).
      </Step>
      <Step title="Install hackrf">
        ```bash
        brew install hackrf
        ```
      </Step>
      <Step title="Verify the device is accessible">
        Plug in your HackRF and run:

        ```bash
        hackrf_info
        ```

        You should see the board ID and serial number printed.
      </Step>
      <Step title="Launch SDRTrunk">
        Open SDRTrunk. The HackRF device should appear in the tuner manager.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Configuration in SDRTrunk

Open the tuner editor from the **Tuners** tab after SDRTrunk detects your HackRF.

### Gain controls

HackRF exposes three independent gain stages:

| Stage | Control | Range | Description |
|-------|---------|-------|-------------|
| **LNA gain** | `SET_LNA_GAIN` | 0, 8, 16, 24, 32, 40 dB | RF front-end low-noise amplifier gain applied before the mixer |
| **VGA gain** | `SET_VGA_GAIN` | 0–62 dB (2 dB steps) | Baseband variable gain amplifier gain applied after down-conversion |
| **Amplifier** | `AMP_ENABLE` | Off / On | Optional 14 dB RF pre-amplifier before the LNA |

<Tip>
  Start with the amplifier off, LNA gain at 16 dB, and VGA gain at 20 dB. Enable the amplifier and increase LNA gain only if signals are too weak.
</Tip>

<Warning>
  Enabling the amplifier when receiving strong signals can cause overloading and intermodulation. Use it only when needed for weak-signal work.
</Warning>

### Other settings

- **Frequency** — center frequency in MHz. SDRTrunk places individual channels within the HackRF's bandwidth around this center.
- **Sample rate** — select from the supported rates above. The baseband filter is set automatically to match.
- **PPM correction** — software frequency correction in parts per million. HackRF's clock accuracy is generally better than RTL-SDR but you can still apply a correction if needed.

## Common issues

<AccordionGroup>
  <Accordion title="HackRF is not detected by SDRTrunk">
    On Windows, ensure the WinUSB driver is installed via Zadig. On Linux, confirm the udev rules are in place and your user is in the `plugdev` group. Only one application can access HackRF at a time — close any other SDR software (GNURadio, SDR#, CubicSDR) before launching SDRTrunk.
  </Accordion>
  <Accordion title="HackRF is in use by another application">
    HackRF can only be used by one application at a time. Ensure no other software is accessing the device before launching SDRTrunk.
  </Accordion>
  <Accordion title="Strong signals cause distortion across the spectrum">
    Reduce LNA gain and disable the front-end amplifier. HackRF has no hardware AGC, so you must manually adjust gain to avoid overloading the front end when strong transmitters are nearby.
  </Accordion>
  <Accordion title="Sample rate changes are not applied">
    Some legacy sample rate values are no longer valid in current SDRTrunk builds. If you load an old configuration with a deprecated rate, SDRTrunk will automatically substitute 5.0 MHz.
  </Accordion>
</AccordionGroup>
